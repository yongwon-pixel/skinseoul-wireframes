트래킹넘버 미입력 리스트 — 신규 / 변경 주석 (admin 신규 페이지)

  1. 0신규 admin 페이지 — 트래킹넘버 없는 주문을 한쪽에 치워두던 수동 프로세스를 대체. 쿠팡은 오더넘버 즉시 생성·트래킹넘버 지연 생성 → 미인식·미입력 건을 모아 추적 — F 트래킹 자동화
  2. 1미인식 제품 풀 — View Orders 바코드 인식 실패 팝업을 통해 전송된 물리 제품 집계, 노란 하이라이트 강조. 스캔 시 수집된 라스트마일 트래킹넘버를 주문번호 좌측에 표시. 주문번호(쿠팡 발주)는 대부분 "–" — 조회 매칭 성공 건은 등록 단계에서 해결되어 풀에 안 옴. 각 항목은 브랜드·제품명·사이즈·바코드·갯수·메모 노출 — F 플로우 5
  3. 2의심 오더 (자동 추정) — 바코드→제품 대조로 이 제품이 포함된 오더가 존재하고 그 오더가 Processing 단계인 경우만 후보로 표시, 복수 가능. 대부분 Naver·공식몰 등 비쿠팡 채널의 JIT 주문(쿠팡 건은 주문번호 조회에서 걸러짐). 등록 시 #unrecognized-tracking 자동 알림에 의심 PIC도 멘션 → 핸들러는 와서 확인만 하면 됨 — 2026-07-23 UX 역전
  4. 3"확인해보기" 버튼 → 매칭 모달(M1) — 후보 제품 라인을 클릭해 매칭(오더 단위 아님). 후보 목록이 "제품 포함 + Processing" 오더 전부라 별도 PIC 검색 없음(여기 없으면 데이터 이상 — 2026-07-23 수동 검색 폴백 제거). 확정 시 트래킹넘버가 그 제품 라인에 등록·풀에서 제거 + 해당 오더에 코멘트 자동 등록(미인식 등록자를 @태그 "오더 매칭했습니다" → Slack 알림). 행 우측 ✕ = 리스트에서 제거(오등록·대응 불필요 건 정리) — F 플로우 2·5
  5. 4Product Name 맨 앞 브랜드명 항상 노출 (View Orders B 결정 상속) — B
  6. 5Comments — 우상단 허브([@ Mentions]+[★ Saved]). @태그 시 Slack 자동 알림, 코멘트 클릭 시 해당 오더 페이지로 이동 — 0. UI 1+2

2026-07-23 단순화 결정: PIC별 그룹 테이블·일괄 처리 바·Slack 알림 칼럼·해결 완료 로그·통합 검색바·페이지 내 등록 버튼과 모달을 모두 제거. 등록은 View Orders 팝업 단일 진입점, 알림은 자동 발송. 매칭 UX 역전: 핸들러가 제품마다 자기 이름을 검색하는 방식(구) → 시스템이 의심 오더·핸들러를 먼저 제시하고 핸들러는 확인만 하는 방식(신). 후보가 "제품 포함+Processing" 전수라 수동 검색 폴백도 제거(2026-07-23).
