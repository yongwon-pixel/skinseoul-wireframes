  2. 2소싱 루트 지정 — 라디오 카드 3종(스마트바이 / 홀세일 / 브랜드 파트너십), 선택 루트에 따라 아래 폼이 바뀜. 배지 색은 View Orders 소싱루트 배지와 동일 규약 — I 입고요청
  3. 3Inbound No. 자동 발번 — 등록 순서대로 YYYYMMDDNNNN(NNNN은 그날 0001~9999) 자동 지정, 수정 불가. 발주가 아닌 입고(파트너십)에도 맞도록 "PO No." 대신 Inbound No.로 통일 — Procurement Hub 시트 A열(12자리)과 동일 체계. PO 매칭 패널은 폐기(2026-07-23) — I + J
  4. 4입고 제품 입력 재설계(2026-07-23) — SKU No. 또는 Product Name으로 검색해 클릭 선택(선택 시 제품명·SKU·Brand 자동 확정), Size 제거·Qty만 입력 — I 입고요청
  5. 5검색 자동완성 — SKU No.·영어 제품명 어느 쪽으로도 카탈로그 검색(View Orders 미인식 모달과 동일 문법), 클릭 선택 시 확정 — I + F 자동완성
  6. 10Unit Cost (KRW) 필수 — FOC(Free of charge) 체크 시 0원 잠금(협찬·무상 증정분) — 시트 Unit Cost 열
  7. 11JIT Price (KRW) 옵셔널 — 스마트바이·홀세일만 입력(파트너십 선택 시 칼럼 미노출) — 시트 JIT Price 열
  8. 12Supplier 필수 — 공급처 기록(시트 Supplier 열 대응) — 시트 연동
  9. 6제품 행 추가 버튼 — 한 요청에 여러 SKU 신청 가능 — I 입고요청
  10. 7송장번호 입력란 — 신청 시점엔 미입력 가능(대시 테두리로 선택 필드 표시), 물량 발송 후 나중에 입력 — I 입고요청
